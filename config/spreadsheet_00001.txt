# SQL取数规则
执行脚本严格按照信息项顺序生成SQL，仅输出纯净的SQL语句，不包含任何格式化标记或额外说明。
请根据目标数据库的特性，使用合适的标识符引用方式（duckdb），并确保字段别名符合标准语法。
特别注意SQL语句的完整性，尤其是关键字如`select`必须准确无误，不能有任何拼写错误或截断。
必须确保所有列名都使用双引号包裹。不必对数据内容进行清洗。
在映射每个字段前，必须确认df中是否存在对应的列，若不存在则使用`null as "字段名"`的形式来填充。
```sql
SELECT
    case
        when '1022社保减员_Sheet1' like '%离职%' or '1022社保减员_Sheet1' like '%减员%' then '离职'
        when '1022社保减员_Sheet1' like '%入职%' or '1022社保减员_Sheet1' like '%增员%' then '入职'
        else ''
    end as "作业",
    "身份证号" as "证件号码",
    "姓名" as "姓名",
    "公司" as "客户名称",
    null as "户籍",
    null as "用工模式",
    "入职日期" as "入职日期",
    null as "合同终止日期",
    "离职日期" as "离职日期",
    null as "社保缴纳城市",
    null as "社保基数",
    null as "公积金基数",
    "社保缴纳截止月" as "社保最后缴纳月",
    "公积金缴纳截止月" as "公积金最后缴纳月",
    null as "成本中心",
    "公司工号" as "内部工号",
    null as "备注"
FROM df
```
