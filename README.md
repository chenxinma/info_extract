# 人力资源入职信息自动化提取工具可行性研究方案

## 项目背景

作为人力资源服务方，我司每日接收大量客户发送的人员入职需求，信息载体包括邮件正文、Excel、Word等附件。当前依赖集约化团队人工提取与整理，存在效率低、易出错、响应慢等问题。客户格式多样，频繁变更，亟需灵活、自主的自动化工具支持。


## 应用场景

1. **多源信息输入**：处理来自客户邮件的正文内容及各类附件（Excel、Word、PDF）。
2. **非结构化文本解析**：从自由格式邮件中提取姓名、身份证号、岗位、入职日期、薪资等关键字段。
3. **结构化表格标准化**：将不同格式的Excel/Word表格统一转换为标准数据模板。
4. **集中批量处理**：支持集约化团队批量导入、一键提取、快速核对与导出。


## 建设目标

1. **提升效率**：实现入职信息自动提取，减少80%人工录入工作量。
2. **保障准确**：通过规则校验与结构化输出，降低人为错误率。
3. **增强灵活性**：提供可视化规则配置界面，支持业务人员自主维护正则规则与大模型提示词，快速适配客户格式变化。
4. **确保安全**：桌面端本地运行，数据不上传，符合个人信息保护要求。
5. **输出标准**：自动生成符合内部系统要求的标准化Excel文件，无缝对接后续流程。


## 方案概述

开发一款桌面应用，集成正则引擎与大模型能力，支持“规则+智能”双模式信息提取。内置规则管理与导出功能，赋能集约化团队高效、精准完成入职信息处理。


## 预期效益

实现入职信息处理的标准化、自动化与自主化，提升服务响应速度与客户满意度，为HR运营数字化奠定基础。

## 架构说明
```mermaid 
%% 本地桌面的结构化信息提取工具

flowchart LR
    id1[\源文件\]
    id1-->extract([info_extract])
    rule["提取规则"]-.->extract
    extract-->f[\入职Excel\]
```

## 功能实现
### 读取邮件正文提取信息 
1. 文件格式 *.eml *.msg，采用email把邮件正文转换成txt文件放入processing目录，txt文件名含创建时间戳(yyyyMMddHHmmss)。

### 使用方法
1. 将需要处理的eml或msg邮件文件放入 `source` 目录
2. 运行命令：`uv run info-extract`
3. 程序会自动处理所有邮件文件，并将提取的正文内容保存为txt文件到 `processing` 目录
4. txt文件名格式为：`{原始文件名}_{时间戳}.txt`

### 依赖说明
- `msg-parser[rtf]`：用于解析Microsoft Outlook的msg文件
- `email`：Python标准库，用于解析eml文件

### 错误处理
- 对于无法用msg-parser处理的msg文件，程序会自动采用回退机制按文本文件处理
- 所有处理过程和错误信息都会记录在日志中
