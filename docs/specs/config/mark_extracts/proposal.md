# 划词信息项标记功能提案

## 背景
目前系统已完成信息项的CURD功能，但在文本信息结构化提取场景中缺少有效的训练数据准备工具。需要开发一个划词信息项标记功能，让用户能够对文本片段进行信息项标注，以便构建训练数据集。

## 目标
开发一个划词信息项标记界面，支持用户：
1. 管理示例文本片段
2. 对文本进行划词标注
3. 为标注信息添加属性
4. 保存和导出标注数据

## 需求详情

### 功能需求

#### 需求1：示例文本管理
系统应允许用户管理用于标注的示例文本片段。
- 用户可以查看已有的示例文本列表
- 用户可以添加新的示例文本
- 用户可以删除不需要的示例文本
- 用户可以选择一个示例文本进行标注

#### 需求2：划词标注功能
系统应提供直观的划词标注功能。
- 用户可以在文本编辑区域内输入或粘贴文本
- 用户可以选择文本中的特定部分
- 系统应提供信息项按钮供用户点击以完成标注
- 标注后的文本应有视觉标识以区分不同信息项

#### 需求3：标注结果展示
系统应清晰展示当前示例的所有标注结果。
- 以表格形式展示标注的文本、信息项类型和相关属性
- 支持查看、编辑和删除已标注的信息
- 支持为选定的标注结果添加自定义属性

#### 需求4：数据持久化
系统应保存用户的标注结果。
- 标注数据应与对应的示例文本关联
- 支持更新已有标注
- 提供统一的数据导出功能

## 技术方案概述
- 前端使用HTML/CSS/JS实现交互界面
- 后端需要提供API来管理示例文本和标注数据
- 使用现有信息项配置作为标注选项的数据源
- 数据模型需包含示例文本、标注记录及其属性信息

## 预期成果
- 一个可用的划词标注界面
- 完善的前后端数据交互逻辑
- 高质量的用户标注体验
- 便于后续训练数据使用的导出功能
