# 提案：添加 PDF 源支持

## 问题陈述
目前，系统仅支持电子邮件（eml/msg）和 Excel 文件作为输入源。许多组织还以 PDF 格式发送人员信息，这需要手动处理。

## 提出的解决方案
添加一个新的源模块来处理 PDF 文件，提取可以输入到现有纯文本提取管道中的文本内容。

## 优势
1. 减少处理 PDF 格式人员信息的手动工作
2. 增加对更多文件格式的自动化覆盖
3. 与现有的提取工作流保持一致

## 实现方法
1. 使用 PyPDF2 或 pdfminer 等库添加 PDF 解析功能
2. 创建遵循现有源模块模式的 PDFReader 类
3. 提取文本内容并保存为处理目录中的 txt 文件
4. 利用现有的 PlainExtractor 进行信息提取

## 资源需求
1. 开发时间：2-3 天
2. 依赖 PDF 解析库
3. 使用各种 PDF 格式进行测试
